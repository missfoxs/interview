1. 数组赋值

   如果数组的下标为字符串或者对象，也是可以的，因为javascript的数组是关联数组，继承自Object。此时数组的长度属性不会发生变化，并且可以使用for...in方法当成对象来遍历。其实，普通的数组也是可以用for...in遍历的。

   ```javascript
   let arr = [];
   let obj = {name: '1'};
   arr[JSON.stringify(obj)] = 1;
   log(arr);   //[ '{"name":"1"}': 1 ]
   log(arr.length);  // 0
   arr[5] = 5;
   log(arr.length);  //6
   for(let key in arr){
       log(arr[key]);
   }
   ```

17. 深浅拷贝

    浅拷贝：如果是数组的话，可以使用concat， slice方法

    对象的浅拷贝：先判断是否为对象，不是直接返回，然后判断是否为数组来定义一个新数组或对象，将原对象的每个属性复制到新对象上（数组的键为对象的键，值为对象的值）

    ```javascript
    function copy(oldObj) {
      if (typeof oldObj !== 'object') return;
      let newObj = Array.isArray(oldObj) ? [] :{};
      for(let key in oldObj) {
        if(oldObj.hasOwnProperty(key)) {
          newObj[key] = oldObj[key];
        }
      }
      return newObj;
    }
    ```

    深拷贝：使用递归，在判断属性时，判断是否为对象，如果是，递归调用即可

    ```javascript
    function copy(oldObj) {
      if (typeof oldObj !== 'object') return;
      let newObj = Array.isArray(oldObj) ? [] :{};
      for(let key in oldObj) {
        if(oldObj.hasOwnProperty(key)) {
          if(typeof oldObj[key] === 'object') {
            newObj[key] = copy(oldObj[key])
          } else {
            newObj[key] = oldObj[key];
          }
        }
      }
      return newObj;
    }
    ```

    还有一个实现深拷贝的方式，即使用JSON.stringify()方法，但是他不会拷贝function对象

19. 同源策略以及实现跨域的方式

    同源主要是浏览器保护数据安全的机制，具体来说，一个源上的文档或者他加载的脚本不可以访问另一个源上的一些数据，比如cookie, localstorage, indexDB, 发送XMLHttpRequest, Fetch请求。当然也有一些方法绕开该限制

    1. 绕开xmlhttprequest和fetch限制的方法(还包括web字体)
       * jsonp
       * cors    Access-Control-Allow-Origin属性
       * websocket
       * 代理
    2. 绕开cookie限制的方法
       * 设置document.domain: 该属性指定cookies对哪个域有效，cookies只会发向该域
    3. 跨窗口通信
       * window.name
       * window.postMessage

16. CORS的实现

    如上所述，cors可以实现xmlhttprequest的跨域访问资源

    发送ajax请求时，其实也是发送XMLHttpRequest请求，该API是浏览器提供的，浏览器会在帮我们做很多事情，如果浏览器发现该请求是跨域的，则会加上origin请求头，相应地，远程服务器可以检查 Origin首部，决定是否接受该请求，如果接受就返回 Access-Control-Allow-Origin 响应首部，如果不同意接受这个请求，那么只要不在响应中包含 Access-Control-Allow-Origin 首部即可。这样，客户端的浏览器就会自动将发出的请求作废。

    另外，cors请求分为简单和非简单两种，如果使用了指定的方法（get, post, head），指定的请求头，那么该请求就是简单请求，否则就是非简单请求，非简单请求需要先发送一次option请求，返回一些具体的信息。具体见：
